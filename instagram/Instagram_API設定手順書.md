# Instagram 自動投稿のための Meta API 設定手順書

AI（Python）からInstagramへ自動投稿を行う場合、公式の「Instagram Graph API」を利用します。このAPIを使うためには、事前にFacebookページとの連携や開発者アプリの作成が必要です。

ここでは、その具体的なステップを解説します。

---

## 1. Facebookページの作成
**個人アカウント（例: pankish0525）から作成して全く問題ありません！**
（Facebookの仕組み上、すべての「ページ」は誰かの「個人アカウント」が管理者として作成するルールになっています。）

1. Facebook（個人アカウント）にログインします。
2. 左側のメニューから **「ページ」** をクリックします。
3. 左上にある **「新しいページを作成」** をクリックします。
4. ページ名（例: `カブ先生のお金の学校`）、カテゴリ（例: `教育サイト` や `金融サービス`）を入力し、**「ページを作成」**を押します。
   * ※このページは連携のシステム上「存在すること」が重要なので、凝ったデザインにする必要はありません。

## 2. Instagramを「プロアカウント」に切り替える
個人のInstagramアカウントはAPI経由で操作できないため、プロフェッショナルアカウント（ビジネス）に切り替えます。

1. スマートフォンのInstagramアプリで「カブ先生」のアカウントを開きます。
2. 右上の設定メニュー（≡） ＞ **「設定とプライバシー」** に進みます。
3. 下の方にある「プロフェッショナルのためのツール」の項目から **「プロアカウントに切り替える」** をタップします。
   * *※もしすでになっている場合は「ビジネスツールと管理」のような表示になっています。*
4. カテゴリを選択し、アカウントの種類で **「ビジネス」** を選びます（クリエイターではなくビジネスを推奨します）。

## 3. FacebookページとInstagramアカウントを連携する
作成したFacebookページとInstagramを紐付けます。

1. 先ほど作成したFacebookページ（パソコン）を開きます。
2. 左のメニューから **「設定」** ＞ **「リンク済みのアカウント」** をクリックします。
3. 「Instagram」を選び、**「アカウントをリンク」** をクリックします。
4. Instagramのログイン画面が出ますので、「カブ先生」のIDとパスワードでログインして連携を完了させます。

## 4. Meta for Developersでアプリを作成する
ここからがシステム用の「鍵（トークン）」を発行する作業です。

1. **[Meta for Developers](https://developers.facebook.com/)** にアクセスし、個人のFacebookアカウントでログイン（登録）します。
2. 右上の「マイアプリ」をクリックし、**「アプリを作成」** をクリックします。
3. ユーザーの目的として **「その他」**（または「ビジネス」）を選択して「次へ」。
4. アプリタイプは **「ビジネス」**（またはコンシューマー）を選択します。
5. アプリ名（例: `KabuPortalInstaBot`）等を入力して **「アプリを作成」** をクリックします。

## 5. APIの使用設定とアクセストークンの発行
1. アプリのダッシュボード画面で、製品を追加する画面から **「Instagram Graph API」** と **「Facebookログイン」**（ビジネス向け）をセットアップします。
2. メニューの「ツール」＞ **「グラフAPIエクスプローラ (Graph API Explorer)」** を開きます。
3. 画面右側の設定で以下を行います：
   * **Facebookアプリ**: 先ほど作ったアプリを選択
   * **ユーザーまたはページ**: 「ページアクセストークンを取得」を選び、作成した「カブ先生のページ」を選択
   * **許可（Permissions）**: 以下の4つを追加（Add）します。
     - `instagram_basic`
     - `instagram_content_publish`
     - `pages_show_list`
     - `pages_read_engagement`
4. **「Generate Access Token（アクセストークンを生成）」** をクリックします。
5. 長い文字列（トークン）が生成されます。これをコピーします。
   * *※このトークンは短期間で切れるため、次の「トークン延長ツール（Access Token Debugger）」で60日用の長期トークンに変換する必要があります。*

*(※このステップ5以降は少し専門的なので、ステップ4まで終わりましたら私に教えてください。長期トークンの発行や環境変数へのセットは、スクリプト等で私がサポートいたします！)*
