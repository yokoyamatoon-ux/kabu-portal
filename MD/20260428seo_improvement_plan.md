## 診断ツールが指摘した優先改善項目

### 🔴 第1優先：H1タグ（効果：大）
各ページにH1タグが1つだけ存在し、対象キーワードを30文字前後で含むこと。現状は複数あるか欠落している可能性がある。

### 🔴 第2優先：見出し階層（効果：大）
H1→H2→H3の階層が正しく構成されていない。H1をスキップしてH2を使う、H2の中にH4が入るなどの問題がある。

### 🔴 第3優先：構造化データ JSON-LD（効果：大）
現スコア **0点**。Article・FAQPage・Person スキーマの実装が必要。  
※ 今回の改修で実装済みのはずだが、ツール上では0点のまま→**デプロイ後の反映待ちか、実装に漏れがある可能性あり**

## 改善プラン（3フェーズ）

### Phase 1：即時対応（〜2週間）
技術的な修正のみ。コンテンツ変更なし。

| # | 作業 | 担当箇所 | 期待効果 |
|---|---|---|---|
| 1 | 各ページのH1タグを1本に統一 | 全固定ページ・記事ページ | 第1優先の解消 |
| 2 | 見出し階層をH1→H2→H3に整理 | 全コラム・マンガページ | 第2優先の解消 |
| 3 | JSON-LDのデプロイ確認・Googleリッチリザルトテストで検証 | col_013・manga/1・about | 構造化データ0点の解消 |
| 4 | サイテーション（外部リンク）の獲得準備 | — | サイテーション0点への着手 |

**H1タグの修正指針**
```
NG：ページ内にH1が複数ある
NG：H1がなくH2から始まる
OK：H1が1本、キーワードを含む30文字前後
例）<h1>NISAだけじゃもったいない！iDeCoって結局なんなの？</h1>
```

**見出し階層の修正指針**
```
NG：H1 → H3（H2をスキップ）
NG：H2の中にH2（同じレベルが入れ子）
OK：H1 → H2 → H3 の順番を守る
```

## 補足：JSON-LD 0点について

今回の改修でJSON-LDを実装済みにもかかわらず0点になっている理由として考えられるのは以下の3つです。

1. **デプロイがまだ済んでいない**（最も可能性が高い）
2. **診断ツールがJavaScript実行をしておらず、RSC Payloadを読んでいない**
3. **JSON-LDの構文エラー**

→ デプロイ後にGoogleリッチリザルトテスト（https://search.google.com/test/rich-results）で `col_013` を検索して確認するのが最優先です。
