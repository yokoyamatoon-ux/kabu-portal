# Antigravity 修正プロンプト
## ニュース取得をTTLキャッシュ方式に変更する

---

## 背景・問題

現在のニュース取得は古いデータが表示され続けている。
Streamlit Cloudはスケジューラが使えないため、
**アクセスのたびにRSSを取得し、6時間キャッシュする**方式に変更する。

---

## 対象ファイル

`modules/news_unit.py`

---

## やること

### ① まず現状のコードを確認する

`news_unit.py` を開いて以下を確認する：
- RSS取得関数がどこにあるか
- `@st.cache_data` がすでに使われているか
- TTL（time_to_live）の設定があるか

---

### ② RSS取得関数に TTL=6時間 のキャッシュを設定する

RSS取得を行っている関数（`fetch_rss` や `get_news` などの名前の関数）に、
以下の形式でデコレータを設定する。

**変更前（キャッシュなし、またはTTLなしの場合）：**
```python
def fetch_rss(url):
    ...
```
または
```python
@st.cache_data
def fetch_rss(url):
    ...
```

**変更後：**
```python
@st.cache_data(ttl=21600)  # 6時間 = 60×60×6 = 21600秒
def fetch_rss(url):
    ...
```

複数のRSS取得関数がある場合は**すべてに** `ttl=21600` を設定する。

---

### ③ カテゴリごとのニュース取得関数にも同様に設定する

カテゴリ別（日本の株価・注目の銘柄・初心者NISA・海外の株価）に
まとめてニュースを取得している関数がある場合も、
同様に `@st.cache_data(ttl=21600)` を設定する。

---

### ④ 手動更新ボタンを追加する（任意だが推奨）

ニュースセクションの上部に「🔄 ニュースを更新」ボタンを追加する。
ボタンを押したときにキャッシュをクリアして再取得する。

```python
if st.button("🔄 ニュースを更新", key="news_refresh_btn"):
    st.cache_data.clear()
    st.rerun()
```

---

### ⑤ 最終取得時刻を表示する

ニュースセクションのタイトル横に、最終取得時刻を表示する。

```python
from datetime import datetime

last_updated = datetime.now().strftime("%m/%d %H:%M")
st.markdown(f'取得時刻: {last_updated}', unsafe_allow_html=True)
```

---

## 変更してはいけないもの

- RSSのURL・カテゴリ構成・バナー画像のパスは変更しない
- カードのデザイン・レイアウトは変更しない
- 「カブ博士に聞く」ボタンの挙動は変更しない

---

## 確認ポイント

1. `news_unit.py` 内のRSS取得関数すべてに `@st.cache_data(ttl=21600)` が付いている
2. サイトを開いたときにRSSが取得され、最新ニュースが表示される
3. 「🔄 ニュースを更新」ボタンを押すとキャッシュがクリアされ再取得される
4. 6時間以内の再アクセスではキャッシュが使われ、余計なリクエストが発生しない
5. カードのデザイン・バナー画像・ボタンに変化がない
